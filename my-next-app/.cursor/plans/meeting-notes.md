## 3D Design 팀 리뷰

- 산업군 메시지보다 **시뮬레이션 방식 자체**를 전면에 세우는 편이 설득력 있음
- 랜딩 대표 기능은 Animation보다 **Simulation Modeling**에 무게를 둬야 함
- 기능 소개는 단순 나열보다 **"복잡한 의상을 안정적으로 만든다"는 결과 중심 메시지**
- Animation을 전면 후킹 포인트로 쓰면 제품 인식이 흐려질 수 있음
- Live Sync, DNA Importer는 상세 페이지용 — 메인 랜딩 첫 줄 메시지 아님
- IA를 **기능별(Features)**로 잡아도 업계 사용자 이해에 무리 없음
- Domain 페이지를 만들더라도 상세 프로세스 정의보다 **"이 산업에서 MD가 들어가는 대표 장면"** 정도만

## CX 팀 리뷰

- 튜토리얼: 유튜브 중심, 웹사이트 배너 등 **진입 초반 노출 구조**
- **학생 플랜**: 6개월 무료 프로모션 검토, 풀피처 + 상업적 사용 불가 명확히 전달
- **플랜 비교표 재도입** — 카드 나열보다 비교 구조 필요
- **고객 유형별 랜딩**: Individual / Enterprise / Academic·Student 분기
- 현지 통화 결제 4개국 도입 검토
- 플랜 페이지 개편 = 콘텐츠 리뉴얼 + **실제 구매 전환 구조**까지

## BD 팀 리뷰

- Domain 페이지: 기능 설명보다 **"이 결과를 만들 수 있다"는 사례 중심**
- Enterprise 페이지: "기능 나열" → **"왜 기업이 선택해야 하는가"** 집중
- 메시지: 신기한 기능보다 **"현업에서 검증된 표준 툴"**
- IA 기능 분류: 내부 용어가 아닌 **3D 디자이너가 바로 이해하는 언어**로 재정비 필요
- Academic 페이지: 기능 상세보다 **취업·업계 연계성 중심**
- 플랜 명칭: 결제 방식 기준 → **사용자군 기준**으로 재설계 (월별/연간/네트워크 → Individual/Enterprise/Academic)
- Enterprise 비교표: 기능 비교 → **관리·지원·운영 편의성 비교**
- 글로벌 = 번역 + **현지 통화·결제수단·가격 노출 정책** 모두 포함
